# PRD V1: 贡献协作规范

## 背景
项目已进入可持续维护阶段，需要让新贡献者在不了解历史会话的前提下，按统一流程完成开发、提测、提 PR 与 Code Review。

## 目标
- 定义统一分支规范、提交规范、PR 提交流程。
- 提供可执行的 Review Checklist，降低回归风险。
- 将规范沉淀为中英文文档，便于开源协作。

## 非目标
- 不引入 CI 平台改造。
- 不引入复杂的发布审批系统。

## 用户故事
- 作为贡献者，我希望知道改动前后必须执行哪些命令。
- 作为 Reviewer，我希望有统一清单来判断是否可合并。
- 作为维护者，我希望减少“代码能跑但不满足质量门禁”的 PR。

## 功能需求
- 贡献文档需包含：
  - 分支命名规则。
  - 提交信息规则。
  - PR 结构与描述模板。
  - 合并前必跑命令清单。
  - Review Checklist（功能、测试、文档、风险、回滚）。
- 在仓库提供默认 PR 模板，减少遗漏项。

## 成功指标
- 新贡献者可按文档在一次提交中通过质量门禁。
- PR 描述结构一致，Review 往返轮次下降。

## 验收标准
- 新增中英文贡献文档。
- 新增 PR 模板。
- README 有贡献入口。
