# PRD V1: 标签与视图（更新版）

## 背景
Nexa 仍以 Conversation 与 Note 为一等实体，并保留“手动标签 + 自动标签 + 未来自然语言视图”的总体方向。为保证 V1 交互聚焦，本阶段不再在主界面暴露独立标签管理区。

## 目标
- 维持标签体系的产品方向与数据模型预留。
- 保留自动标签流水线可继续演进的能力。
- 不阻塞 V2 视图规则与自然语言聚合能力落地。

## V1 边界（本次更新）
- 主界面移除标签管理占位区，仅保留设置入口。
- 不在侧栏展示手动/系统标签列表。
- 系统标签“仅支持删除”的产品规则继续有效，但入口延后到后续设置/详情页。

## 非目标
- V1 不提供完整标签后台管理页。
- V1 不提供视图（Project）UI。
- V1 不支持 Workspace。

## 用户故事
- 我现在更关注对话与分叉体验，不希望左侧充斥尚未完成的标签管理占位。
- 我希望未来仍能在设置或专门页面中管理自动标签策略和结果。

## 功能需求（V1）
- 数据层
  - 标签实体与关系结构继续保留。
  - 自动标签任务链路保持可用并可被后续页面调用。
- UI 层
  - 侧栏仅保留设置入口，不展示标签管理表单。
- 架构层
  - 预留“设置页/详情页”挂载标签管理入口。
  - 保留 `rule_nl + rule_json` 的自然语言视图预留设计。

## 成功指标
- V1 主流程（聊天/分叉/图谱）不受标签 UI 占位干扰。
- 标签相关代码在无 UI 暴露场景下仍可被测试覆盖与调用。

## 风险
- 用户可能误解为标签能力被取消。

## 缓解
- 在文档与 V2 规划中明确：标签能力为“入口后置”，并非“功能删除”。

## 未决问题
- 标签管理入口优先放在设置页还是内容详情页。
