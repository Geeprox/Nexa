# PRD V1: 交接与维护手册

## 背景
Nexa 已形成可运行的 V1 能力（分叉对话、图谱、搜索、标签、后台任务、macOS 壳层）。项目目标是开源长期演进，因此需要让新维护者在脱离当前会话后，仍能独立完成开发、测试、发布与排障。

## 目标
- 提供一份可执行的 V1 交接文档（中文默认 + 英文切换）。
- 提供明确的维护手册，覆盖日常开发、质量门禁、发布构建、常见故障处理。
- 将“做什么、在哪里改、怎么验证”固化成标准流程，减少知识只存在于会话的问题。

## 非目标
- 不在本阶段引入 CI/CD 平台集成。
- 不在本阶段引入自动化签名/公证流水线。
- 不扩展新的产品功能（聚焦交接与维护）。

## 用户故事
- 作为新维护者，我希望 30 分钟内在本地跑起 Web + macOS 应用。
- 作为发布维护者，我希望按文档可重复产出可安装的 macOS `.app`，并拿到标准产物清单。
- 作为贡献者，我希望知道修改后必须运行哪些命令，才能避免功能回归。

## 功能需求
- 文档提供：
  - 快速启动流程（开发、测试、构建、桌面运行）。
  - 模块架构与关键代码入口索引。
  - 发布流程与产物清单文件说明。
  - 故障排查手册（网络代理、Tauri 打包、白屏与资源错误等）。
  - V2 扩展建议与维护边界。
- 文档入口需在 README 中可见。

## 成功指标
- 新成员按文档执行，`npm test`、`npm run build`、`cargo build` 可一次通过。
- 发布脚本可输出 `dist/releases/macos/latest.json`。
- 关键维护操作（改配置、改路由、改标签策略）都有明确路径指引。

## 风险
- 文档和实现偏离（需要维护机制约束）。
- 开发环境差异导致本地打包行为不同（尤其 `.dmg`）。

## 验收标准
- 新增中英文 handoff 文档。
- README 中新增 handoff 入口。
- 计划文件中 M5.3 标记完成并给出下一步任务。
