# PRD V1: 高亮笔记与关键词搜索（更新版）

## 背景
V1 当前重点已转向“聊天输入-流式-分叉-图谱切换”闭环。搜索与笔记能力继续保留方向，但入口与交互节奏需避免与主流程冲突。

## 目标
- 保留笔记与关键词搜索能力的架构与数据层。
- 统一左侧导航，仅保留一个搜索入口（`搜索` 菜单项），移除重复搜索输入框。
- 通过“全部笔记”导航保留笔记一级入口语义。

## 非目标
- V1 不做复杂搜索面板交互优化。
- V1 不做语义向量检索。

## 用户故事
- 我希望左侧结构简洁，不要重复出现两个搜索入口。
- 我希望未来点击“搜索”后进入统一搜索视图，而不是在侧栏顶部堆输入框。

## 功能需求（V1 更新后）
- 导航层
  - 侧栏保留 `搜索` 与 `全部笔记` 导航项。
  - 移除侧栏顶部冗余搜索输入框。
- 数据层
  - 关键词搜索相关存储和查询能力保留。
  - 笔记实体与来源引用字段保留。

## 交互规划
- 当前：`搜索` 菜单项作为入口占位，避免重复控件。
- 后续：进入独立搜索页面/面板，承载输入、结果列表和定位跳转。

## 风险
- 搜索入口可见但能力入口尚未完整，可能产生预期落差。

## 缓解
- 在 V2 迭代中补齐独立搜索页，文档中明确“入口已保留，交互待增强”。
