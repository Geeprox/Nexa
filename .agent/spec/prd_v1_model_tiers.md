# PRD V1: 分层模型配置与任务委派

## 背景
不同任务对成本与能力需求不同，需要将高价值任务交给高能力模型，后台轻量任务交给低成本模型。

## 目标
- 支持高/中/低三层模型配置。
- 不同任务类型可配置默认模型等级。
- 用户可在设置页面修改模型及 API 参数。

## 非目标
- 模型市场与自动推荐。
- 多用户共享模型配置。

## 用户故事
- 我希望聊天用高质量模型，但摘要用低成本模型。
- 我希望在设置页快速切换模型等级。

## 功能需求
- 三层模型配置: high, mid, low。
- 每层模型可配置 base_url, api_key, model_id。
- 任务类型 -> 模型等级映射。
 - 自动标签任务默认使用 low 等级。

## 交互流程
- 进入设置页 -> 配置模型等级 -> 保存。
- 聊天或后台任务触发 -> 选择对应模型。

## 成功指标
- 设置变更后立即生效。

## 风险
- 不同供应商协议差异。

## 未决问题
- 是否支持“高/低”简化模式。
